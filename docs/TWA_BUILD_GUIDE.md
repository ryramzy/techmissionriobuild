# TechMission Rio — TWA Build Guide

This document provides a step-by-step developer guide on compiling the TechMission Rio Progressive Web App (PWA) into an Android Trusted Web Activity (TWA) package. This is required for listing the application on the Google Play Store.

---

## Prerequisites
Before starting, ensure your local development workstation is equipped with:
- **Node.js**: Version 20 or higher.
- **Java JDK**: JDK 11 or higher (required by Bubblewrap for Android compilation).
- **Android SDK**: Install command-line tools or download Android Studio to manage platform build tools.
- **Google Play Console Account**: An active developer account with an application ready for publishing.

---

## Step 1: Install Bubblewrap CLI
Bubblewrap is the official CLI tool provided by Google to generate and compile Android packages from web manifests. Install it globally:

```bash
npm install -g @bubblewrap/cli
```

---

## Step 2: Initialize the TWA Project
To set up the Android project, run the initialization wizard pointing to our production manifest:

```bash
bubblewrap init --manifest https://techmissionrio.org/manifest.json
```

During initialization, configure the prompt parameters:
1. **Package Name**: Use `org.techmissionrio.app` (this matches our production domains and is permanent once submitted).
2. **Signing Keystore**: When asked, select to generate a new signing keystore.
3. **Password Configuration**: Generate and save a secure password using a password manager.
4. **Keystore Details**: Fill out organization details and note where the keystore is generated.
5. **SHA-256 Fingerprint**: At the end of the initialization flow, Bubblewrap will output the SHA-256 fingerprint representing the signing certificate. **Copy this fingerprint**; you will need it in Step 3.

---

## Step 3: Update Domain Association (assetlinks.json)
To allow the Android app to open in full-screen without displaying the browser address bar, we must establish trust.

1. Open [public/.well-known/assetlinks.json](file:///c:/Users/DELL%20I5%20DE%208%C2%BA/Soft%20Dev/techmissionriobuild/public/.well-known/assetlinks.json).
2. Replace the placeholder sentinel `"REPLACE_WITH_SHA256_FROM_PLAY_CONSOLE"` with the copied SHA-256 fingerprint generated by Bubblewrap.
   - *Example Format*: `["AB:CD:EF:01:23:45:67:89:AB:CD:EF:01:23:45:67:89:AB:CD:EF:01:23:45:67:89:AB:CD:EF:01:23:45:67:89"]` (colon-separated hexadecimal octets).
3. Commit and deploy this change to production.
4. Verify the link is publicly accessible and validates correctly:
   - URL: `https://techmissionrio.org/.well-known/assetlinks.json`
   - Validator tool: [Google Digital Asset Links Tool](https://developers.google.com/digital-asset-links/tools/generator) (enter package name `org.techmissionrio.app` and signature fingerprint to confirm).

---

## Step 4: Build APK (Local Testing) & AAB (Play Store Submission)
With signing keys and asset associations established, run the compile step inside the project directory:

### For Local Testing (APK):
```bash
bubblewrap build
```
This builds a signed APK file named `app-release-signed.apk` in your build output directory. Install it on an Android device or emulator with:
```bash
adb install android-build/app-release-signed.apk
```
Confirm the app opens in clean full-screen mode without any browser URL bar displays.

### For Play Store Upload (AAB):
Google Play Console now requires new applications to upload in the **Android App Bundle (AAB)** format instead of APKs.
1. Run `bubblewrap build` to verify the `.aab` file is outputted in the `android-build/` subdirectory.
2. Store this `.aab` file safely for Play Console upload.

---

## Step 5: Play Console Submission Checklist
Log into your Google Play Console and complete the release checklist:

- [ ] **Create App**: Set up the app under the package name `org.techmissionrio.app`.
- [ ] **Upload Bundle**: Upload the generated `.aab` bundle to the Internal Testing track first.
- [ ] **Configure Default Language (PT-BR)**:
  - **Title**: `TechMission Rio` (30 characters).
  - **Short Description**: `Conectando jovens do Rio com educação em tecnologia e mentoria global.` (80 characters).
  - **Full Description**: Focus on our donation connection model, campus partner alignment, student profiles, and mentor messaging portals.
- [ ] **Configure Additional Languages**: Add the English (EN-US) translation profile.
- [ ] **Privacy Policy**: Set target to `https://techmissionrio.org/privacy`.
- [ ] **Content Rating**: Complete the rating survey under the Education category.
- [ ] **App Category**: Set category to `Education`.
- [ ] **Store Graphics**:
  - App icon (512×512px PNG).
  - Feature graphic (1024×500px PNG featuring brand colors).
  - Minimum of 2 phone screenshots (aspect ratio 9:16 or similar).
- [ ] **Release**: Submit the track release for review.

---

## ⚠️ Keystore Security Warning
- **DO NOT commit** the `.keystore` signing keys or raw password files to your public git repository. Keep them secure locally or store them in a secure secret manager.
- If you lose the keystore file or forget the password, you will be unable to release updates to the Google Play Store under this package name. Keep backups in safe offline/cloud spaces.
