# Frontend Standards

## Stack
- Next.js 16.2.1
- Tailwind CSS 3.4.1
- TypeScript
- Vercel (current)

## Future
- Cloud Run migration ready
- PWA-first architecture

## UI Rules
- Reusable components only
- No inline styles unless necessary
- Accessibility required
- Mobile-first design

## Performance
- Next.js Image optimization
- Minimal bundle size
- Lazy loading components
- PWA optimization

## Code Standards
- TypeScript strict mode
- ESLint + Prettier
- Semantic HTML5
- ARIA labels

## Component Structure
```
components/
├── Navigation/
├── Button/
├── Card/
└── UI/
```

## Styling Approach
- Tailwind-first
- Custom TMR utilities
- CSS-in-JS only when necessary
- Responsive design

## Testing
- Playwright for E2E
- Unit tests for utilities
- Accessibility testing
- Performance monitoring
